# HexaIQ MVP Execution Roadmap

작성일: 2026-06-29

## 1. 추천 개발 전략

Flutter 단일 코드베이스로 Android, iOS, Phone, Tablet/iPad를 동시에 지원한다. 백엔드는 FastAPI + PostgreSQL + Redis로 구성하고, 문제 생성/채점/결제/광고 검증은 서버 중심으로 처리한다.

초기 목표는 "Basic 검사 -> AI 스타일 리포트 -> 성장 기록"까지 완성해 앱스토어/플레이스토어 베타 테스트가 가능한 수준을 만드는 것이다.

## 2. Phase 0: 기반 정리

목표:

- 제품 범위 고정
- 폼팩터 기준 확정
- Flutter/FastAPI 프로젝트 생성
- OpenAPI 초안 작성

완료 조건:

- Android/iOS 빌드가 각각 성공
- Phone/Tablet 레이아웃 샘플 화면 확인
- `/api/v1/health` 동작
- 로컬 PostgreSQL/Redis 실행

## 3. Phase 1: 계정과 프로필

기능:

- Firebase Auth 로그인
- Guest 모드
- 프로필 생성/수정/삭제
- 기기당 최대 3개 프로필 제한
- 클라우드 동기화 기본 구조

주요 화면:

- Splash
- Onboarding
- Profile Select
- Profile Create/Edit
- Settings

완료 조건:

- 프로필 4개 생성 시 서버에서 차단
- 로그아웃/재로그인 후 프로필 복원
- Phone/Tablet 모두 화면 깨짐 없음

## 4. Phase 2: Basic 검사

기능:

- Basic 검사 시작
- 6영역 x 5문항
- 문제 생성 엔진 v1
- 응답 저장
- 진행률 표시
- 검사 완료 처리

문제 엔진 v1 범위:

- 각 영역 최소 3개 유형부터 시작
- Level 1~6 우선 지원
- seed 기반 문제 생성
- 정답/오답 선택지 생성
- 최근 90일 중복 방지 로직

완료 조건:

- Basic 검사 30문항 완료 가능
- 중간 이탈 후 재개 가능
- 응답 로그와 풀이 시간이 서버에 저장
- 동일 프로필에 같은 seed 재사용 방지

## 5. Phase 3: 리포트와 성장 기록

기능:

- 영역별 점수 계산
- 육각형 그래프
- 백분위 placeholder 또는 내부 기준값
- 강점/훈련 방향 문장 생성
- 성장 히스토리 저장
- 최근 검사 요약

완료 조건:

- 검사 완료 후 리포트 화면 표시
- 리포트 문구가 진단처럼 단정하지 않음
- 성장 대시보드에서 이전 검사와 비교 가능
- Tablet에서 리포트 2열 보기 지원

## 6. Phase 4: 광고와 보상

기능:

- AdMob 보상형 광고
- Basic 결과 전 광고 2회
- 광고 완료 서버 검증
- 광고 실패/스킵/네트워크 오류 처리
- 리뷰 보상 쿠폰 구조

완료 조건:

- 광고 완료 전 리포트 접근 제한
- 클라이언트 조작만으로 보상 처리 불가
- 광고 실패 시 재시도/대체 안내 제공

## 7. Phase 5: 베타 출시 준비

기능:

- 앱 아이콘/스플래시
- 앱스토어/플레이스토어 메타데이터
- 개인정보 처리방침/이용약관 링크
- Firebase Crashlytics
- Analytics 이벤트
- 내부 테스트 빌드

필수 이벤트:

- app_open
- profile_created
- test_started
- question_answered
- test_completed
- report_viewed
- ad_started
- ad_completed
- ad_failed

완료 조건:

- Google Play Internal Testing 배포
- TestFlight 배포
- 주요 기기 크기에서 UI 검수 완료
- 개인정보/아동 대상 표현 검수 완료

## 8. Phase 6: Advanced와 Professional

Advanced:

- 120문항
- 영역 완료마다 광고 1회
- 영역별 상세 리포트 강화

Professional:

- 240문항
- CAT 적용
- USD 5 결제
- 광고 없음
- PDF 리포트
- 영수증 서버 검증

완료 조건:

- Google Play Billing 검증
- Apple In-App Purchase 검증
- 결제 복원 지원
- PDF 다운로드/공유 가능

## 9. Phase 7: 관리자와 운영

관리자 기능:

- 문제 템플릿 관리
- 난이도 관리
- 정답률/풀이시간/변별도 확인
- 광고 완료율 확인
- 결제 전환율 확인
- 리포트 문구 관리
- 추천 규칙 관리

완료 조건:

- 문제 템플릿 활성/비활성 가능
- 난이도 보정 후보 확인
- 매출/광고/검사 완료율 대시보드 확인

## 11. 핵심 리스크

문제 품질:

- 영역별 템플릿 수가 부족하면 반복감이 커진다.
- 초기에는 영역당 3~5개 유형으로 MVP를 만들되, 출시 전 영역당 최소 20개 유형 확보가 필요하다.

점수 신뢰도:

- 백분위와 표준점수는 실제 표본 데이터 없이 확정적으로 말하면 위험하다.
- 초기에는 "내부 기준" 또는 "참고용 지표"로 표시한다.

광고/결제:

- 클라이언트 완료 이벤트만 믿으면 우회 가능하다.
- 서버 검증과 구매 복원 처리가 필수다.

아동/청소년 표현:

- 사용자에게 낙인을 주는 문구를 금지한다.
- 부모 모드에서도 자녀 간 우열 비교를 피한다.

Tablet UX:

- 단순 확대가 아니라 two-pane 레이아웃을 제공해야 한다.
- 특히 리포트와 성장 화면은 태블릿에서 정보 밀도를 높인다.

## 12. 베타 성공 기준

- Basic 검사 완료율 60% 이상
- 광고 완료율 70% 이상
- 리포트 조회율 80% 이상
- 7일 재방문율 20% 이상
- 앱 크래시 없는 세션 99% 이상
- 주요 Phone/Tablet 해상도에서 텍스트 겹침 없음
